# 첫수업 MVP 브리프

## 한 줄 정의
첫수업은 학부모가 아이에게 맞는 체험수업을 탐색하고 신청한 뒤,
학원/선생님이 체험 신청을 접수하고 일정을 확정하여 운영할 수 있는 플랫폼이다.

## 목표
- 학부모가 모바일에서 쉽게 체험수업을 찾고 신청할 수 있게 한다.
- 학원/선생님이 신청을 놓치지 않고 일정 확정까지 처리할 수 있게 한다.
- 체험 신청 데이터가 사용자 계정과 연결되도록 회원가입 구조를 갖춘다.

## 핵심 사용자
- parent: 5세~13세 자녀를 둔 학부모
- teacher: 선생님(운영 주체)

## 핵심 원칙
1. 리스트/상세는 로그인 없이 접근 가능
2. 신청과 내 신청내역은 로그인 필요
3. 공개 회원가입은 parent만 허용
4. teacher는 초대/수동 생성 기반 계정만 허용
5. `/studio/*`는 teacher 전용
6. 모바일 우선은 학부모, 데스크톱 우선은 teacher(studio)
7. Figma가 UI 최종 기준
8. MVP는 신청과 운영 흐름 검증에 집중

## 기술 방향
- Frontend: Next.js
- Runtime: Node.js
- Backend/Auth/DB: Supabase
- Deployment: Vercel + Supabase

## 성공 기준
- 학부모가 리스트 → 상세 → 로그인 → 신청까지 완료할 수 있다.
- teacher가 신청 목록 → 신청 상세 → 일정 확정까지 처리할 수 있다.
- 신청 데이터가 사용자와 연결되어 내 신청내역에서 보인다.
