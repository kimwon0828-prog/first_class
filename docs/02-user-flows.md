# 사용자 흐름

## 학부모 흐름
1. 프로그램 리스트 진입
2. 과목/지역/연령 기준으로 탐색
3. 프로그램 상세 확인
4. 체험수업 또는 레벨테스트 신청 버튼 클릭
5. 로그인 또는 회원가입
6. 신청서 작성
7. 신청 완료
8. 마이페이지(`/my`)에서 신청 현황 요약 확인
9. 자녀 관리(`/my/children`)에서 자녀 프로필 등록/수정
10. 내 신청내역(`/my/applications`)에서 상태 확인

## 선생님(studio 운영) 흐름
1. 초대/수동 생성된 teacher 계정으로 로그인
2. studio 대시보드에서 오늘 예약/처리 필요 건 확인
3. 프로그램 관리에서 `trial_class` 또는 `level_test` 프로그램 등록
4. 일정 관리에서 class-linked schedule 생성
5. 체험 신청함에서 신규 신청 확인
6. 신청 상세 확인 및 상태 변경
7. 후속 결과/상담/등록 전환은 후속 phase에서 확장

## 인증 정책 메모
- 공개 회원가입(`/auth/sign-up`)은 parent만 가능
- teacher 공개 회원가입은 허용하지 않음
- `/studio/*` 접근 주체는 teacher로 제한

## 상태 설계 메모
- 현재 신청 상태는 `new -> reviewing -> confirmed -> completed`와 `canceled`를 유지
- 노쇼, 결과입력, 상담필요, 상담완료, 등록완료, 미등록은 후속 phase에서 별도 축으로 분리
